Stay on the cutting edge with Docker 1.6 and test drive it on your Raspberry Pi today

Tue, Mar 31, 2015

If you like to stay on the cutting edge of things as much as we do you will be happy to hear that there is an easy way to test drive Docker 1.6 RC2 on your Raspberry Pi.

What do you get with Docker 1.6?

Docker 1.6 will bring a couple of new features, changes and bugfixes.

The ones that stand out the most are

  • syslog support for the Docker logs
  • setting of ulimits for container
  • building images from an image id
  • pulling images based on id
  • Windows client support (Kitematic - I finally see it coming together!)
  • Labels for container and images
  • ability to set constraints on images - memory, etc.

Find out more details in the changelog yourself.

The only prerequisite to get started is that you are using our Hypriot SD card image for the Rasperry Pi. We published a new version just yesterday and you can the blog post about it here: HypriotOS: Back Again and Better then Ever.

Get it while it is still hot

docker-hypriot_1.6.0-rc2-1_armhf.deb (~ 6MB)
SHA256 checksum

All you have to do is to start up your Pi with our SD card image, download the prepared Docker package and install it with dpkg.

HypriotOS: root@black-pearl in ~
$ wget http://downloads.hypriot.com/docker-hypriot_1.6.0-rc2-1_armhf.deb

HypriotOS: root@black-pearl in ~
$ dpkg -i docker-hypriot_1.6.0-rc2-1_armhf.deb

Afterwards you should be able to quench your curiosity by playing with this shiny new toy. This is not yet a stable release so let us know what you think! And by the way - you can find older versions of Docker as Debian packages in our Download-Area. Installing a new version replaces the existing one and thus makes it possible to switch versions without breaking a sweat.

Have fun.
